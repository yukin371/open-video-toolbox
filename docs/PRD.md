# PRD

## 1. 产品定位

Open Video Toolbox 是一个面向个人创作者、自动化脚本和外部 AI 代理的 CLI 媒体处理与简易剪辑工具箱。

它服务的不是“只会压制”的单点工具，而是以下稳定且真实的需求：

- 让外部 AI 或脚本先拿到标准化媒体信息
- 通过结构化 CLI 命令完成转码、导出和后续可扩展的简易剪辑动作
- 保留可追踪、可重跑、可导出的命令与中间产物
- 支持“AI 先生成剪辑计划，人工再做二次修正”的工作流
- 后续支持 CLI 与可选 Desktop 共用同一套核心工作流

## 2. 目标用户

- 希望用命令行自动化批处理媒体文件的技术型用户
- 需要让 Claude、Codex 等外部 AI 代理调用本地媒体命令的人
- 需要在不内置 AI 的前提下整合媒体处理工作流的人
- 需要保留原始命令、日志和中间结果以便排查问题的维护者

## 3. 产品目标

### 必须达成

- 用现代可维护架构承接“媒体处理 + 可编排 CLI”工作流
- 让探测、规划、执行以及后续剪辑子命令都可观察、可测试、可重放
- 让 CLI 成为第一验证入口，供外部 AI 代理稳定调用
- 软件内不直接承载 AI 推理或供应商耦合
- 让 AI 生成的剪辑计划能够被人工二次修正，而不是只能整段重跑

### 明确不做

- 不追求完整复刻旧工具所有界面和隐式行为
- 不在第一阶段做复杂非线性时间线编辑器
- 不把项目做成仅仅“包一层 ffmpeg 参数输入框”
- 不在仓库内部接 Claude、Codex、LLM API 或其他 AI provider
- 不在 MVP 阶段做完整多轨时间线 GUI 编辑器

## 4. MVP 范围

### 功能范围

- 媒体信息探测
- 单文件与批量任务创建
- 传统转码/导出能力
- 结构化 JSON 输出
- 视频剪切、片段拼接
- 音频分离与音轨提取
- 字幕生成与字幕烧录/外挂导出
- BGM / 多轨音频混音基础能力
- 鼓点 / 节拍点导出，便于卡点剪辑
- `edit.json` 剪辑计划生成、读取和执行
- 日志查看
- 生成可复制的命令预览和可审计的中间产物

### 非功能要求

- 支持长时间批任务执行
- 崩溃后能定位问题
- 日志必须保留外部工具原始输出
- 任务定义与命令计划支持序列化，便于回放与调试
- CLI 输出必须足够稳定，便于外部 AI 代理消费
- 剪辑计划必须是文本可编辑格式，便于人工二次修正

## 5. 核心用户故事

- 作为技术用户，我希望把 `ffprobe`、`ffmpeg` 和后续编辑命令串成一个可重跑的 CLI 流水线。
- 作为外部 AI 代理使用者，我希望 Claude、Codex 能稳定读取 JSON 输出并决定下一步命令。
- 作为创作者，我希望外部 AI 先给出剪辑计划，但我能手工调整入点、出点、片段顺序和 BGM 区间，而不是只能全盘接受。
- 作为做短视频的人，我希望拿到节拍点和鼓点信息，方便卡点剪辑。
- 作为维护者，我希望命令构建逻辑和结构化输出都能被测试覆盖，而不是只能手工点 UI。
- 作为后续桌面端用户，我希望 CLI 已经验证过的数据模型和步骤能够直接复用，而不是再造一套业务逻辑。

## 6. 成功标准

- 在 CLI 中跑通从探测到规划再到执行的闭环
- 结构化输出足以让外部 AI 代理稳定调用
- 传统导出能力仍能复用现有 `Execution` 层
- `edit.json` 足以表达 AI 生成结果和人工二次修正结果
- 新贡献者可以通过文档快速理解媒体层、执行层和 CLI 层边界
