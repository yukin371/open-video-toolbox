# AGENTS

本文件定义仓库内 AI Agent 与自动化工具的协作约束。

## 目标

- 保持项目以可维护性优先，而不是短期堆功能。
- 避免把 GUI 逻辑、任务编排、编码器参数拼接写成一团。
- 优先沉淀模型、边界和测试。

## 修改前必读顺序

1. `README.md`
2. `docs/PROJECT_PROFILE.md`
3. `docs/ARCHITECTURE_GUARDRAILS.md`
4. 目标目录下的 `MODULE.md`
5. 与当前任务直接相关的代码、测试和 `docs/roadmap.md`

如果上述文档缺失或与代码不一致，先补文档，再继续实施。

## 工作方式

- 任何新增功能都先判断属于 `Core`、`Cli` 还是 `Desktop`。
- 涉及编码参数的修改时，优先补模型测试或命令快照测试。
- 未经明确要求，不要为了“兼容旧工具”复制不透明行为。
- 任何外部工具调用都必须保留完整日志与错误输出。
- 新增 shared helper / utility / service / adapter 前，先搜索现有实现；能复用则扩展，不能复用要写明原因和 owner。

## 大改前必须先输出边界摘要

在非平凡改动前，必须先输出以下摘要，再开始编辑：

```text
目标模块：
现有 owner：
影响面：
计划改动：
验证方式：
需要同步的文档：
```

高风险改动还必须补充：

```text
架构风险：
重复实现风险：
回滚路径：
```

## 模块边界

- `OpenVideoToolbox.Core`
  - 负责任务模型、预设模型、命令构建、外部工具抽象、日志模型
- `OpenVideoToolbox.Cli`
  - 负责脚本化入口、调试入口、命令预览与最小可运行验证
- `OpenVideoToolbox.Desktop`
  - 负责桌面交互、任务列表、预设编辑和日志查看

## 完成要求

- 任务不算完成，直到改动已落地，且验证已执行或明确阻塞。
- 完成后必须输出验证结果和文档同步结果。
- 发现会形成重复 owner 或边界漂移的实现时，先收敛设计，再继续编码。

## 必须停下并询问的情况

- 无法确认 canonical owner，新增实现可能与现有模块职责重叠。
- 需要修改产品边界、技术路线或长期模块边界，但现有文档没有明确决策。
- 发现仓库中已有未解释的冲突性改动，且它会直接影响当前任务。
- 需要引入新的外部运行时依赖、持久化形态或跨进程通信方式。

## 禁止事项

- 不要在 UI 层直接拼接 `ffmpeg` 命令。
- 不要把磁盘 IO、进程调用、媒体探测散落在多个模块里。
- 不要引入无法替代的闭源运行时依赖。
- 不要为了“快”跳过文档更新。

## 文档同步

以下内容有变化时，必须同步文档：

- 产品边界变化：更新 `docs/PRD.md`
- 技术路线变化：更新 `docs/tech-stack.md` 或 `docs/architecture.md`
- 仓库协作规则变化：更新 `AGENTS.md`
- 当前交付目标或活跃工作面变化：更新 `docs/roadmap.md`
- 模块职责、owner、不变量变化：更新对应目录下的 `MODULE.md`
