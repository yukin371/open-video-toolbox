# PRD

## 1. 产品定位

Open Video Toolbox 是一个面向个人创作者、字幕组、压制爱好者和轻量工作室的本地媒体处理工作台。

它服务的不是“最新潮的视频平台工作流”，而是以下稳定且真实的需求：

- 批量转码与压制
- 基于预设重复执行任务
- 字幕、音频、封装等多步骤串联
- 失败可追踪、任务可重跑、命令可导出

## 2. 目标用户

- 熟悉“小丸工具箱”或同类桌面压制工具的用户
- 不想直接手写复杂 `ffmpeg` 参数，但希望保留可控性的人
- 需要处理批任务、追日志、管理预设的用户

## 3. 产品目标

### 必须达成

- 用现代可维护架构替代旧式耦合 GUI 工具
- 让任务定义与执行过程可观察、可测试、可重放
- 支持从 GUI 与 CLI 两种入口驱动同一套核心能力

### 明确不做

- 不追求完整复刻旧工具所有界面和隐式行为
- 不在第一阶段支持所有编码器和滤镜能力
- 不把项目做成仅仅“包一层 ffmpeg 参数输入框”

## 4. MVP 范围

### 功能范围

- 媒体信息探测
- 单文件与批量任务创建
- 视频编码参数基础配置
- 音频处理基础配置
- 字幕/封装基础支持
- 任务执行、取消、失败重试
- 日志查看
- 预设导入导出
- 生成可复制的命令预览

### 非功能要求

- 支持长时间批任务执行
- 崩溃后能定位问题
- 日志必须保留外部工具原始输出
- 任务定义支持序列化，便于回放与调试

## 5. 核心用户故事

- 作为压制用户，我希望把一批视频拖进来并套用同一预设，然后直接开始处理。
- 作为调参用户，我希望看到最终生成的命令，而不是只看一堆抽象选项。
- 作为维护者，我希望命令构建逻辑能被测试覆盖，而不是只能点 UI 手工验证。
- 作为兼容旧工具的迁移用户，我希望预设语义尽量稳定，但不接受隐藏副作用。

## 6. 成功标准

- 在 CLI 中完整跑通从探测到命令生成再到执行的闭环
- 桌面端能稳定创建、排队和执行批任务
- 预设模型能够独立演进，不与 UI 控件耦合
- 新贡献者可以通过文档快速理解项目边界和结构
