# Roadmap

最后更新：2026-04-15

本文件用于跟踪项目阶段进度，不只是列出计划。

状态定义：

- `done`：本阶段目标已完成，且已有代码或验证支撑
- `in_progress`：本阶段已开始，但还未达到可验收状态
- `next`：下一阶段的默认开发重点
- `later`：明确在后续阶段处理，当前不抢做

## 当前总览

| Phase | 名称 | 状态 | 当前结论 |
| --- | --- | --- | --- |
| 0 | 仓库初始化 | `done` | 仓库、文档、解决方案骨架已建立 |
| 1 | 核心模型 | `done` | 核心领域模型、序列化结构、基础测试已完成 |
| 2 | 外部工具适配 | `done` | `ffprobe` 解析、`ffmpeg` 命令构建、进程执行器已完成 |
| 3 | CLI 原型 | `done` | `presets` / `probe` / `plan` / `run` 命令面已建立 |
| 4 | Desktop MVP | `next` | 下一阶段重点，尚未开始 UI 集成 |
| 5 | 兼容迁移 | `later` | 等 Desktop MVP 稳定后再进入 |

## 下一阶段

### Phase 4: Desktop MVP

状态：`next`

目标：

- 文件导入
- 媒体信息展示
- 预设选择
- 任务列表
- 执行进度
- 日志查看

建议拆分：

1. 建立桌面应用外壳与导航结构
2. 接入媒体探测服务
3. 接入任务规划和执行服务
4. 做任务列表与日志面板

阶段完成标准：

- 桌面端可导入文件并展示基础媒体信息
- 可选择内置预设并生成任务
- 可执行任务并查看日志
- 与 CLI 共享同一套 `Core` 服务，不重复实现业务逻辑

## 后续阶段
